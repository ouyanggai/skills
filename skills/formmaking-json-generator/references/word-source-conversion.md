# Word/截图来源转换规则

当需求来源是 Word、截图、PDF 截图或“照这个表做”时，先把它当作制式表单，而不是普通字段清单。

## 1. 先做表格台账

- 如果输入是 `.docx`，先运行 `scripts/inspect_docx_tables.py <docx>`，提取真实表格的行、列、合并单元格、列宽、底色。
- 如果只有截图，人工建立同样的台账：标题、主表宽度、每行单元格数量、合并关系、固定分区标题、重复区块、审批意见区。
- 生成 JSON 前必须先确定主表列数。不要为了“方便写 JSON”把 Word 的 8/9 列结构简化成 4/6 列；列数一旦简化，`colspan`、控件宽度和整体轮廓很容易失真。
- Word 的 `tblGrid` 宽度用来推导 `headerRow[].options.width`：标签列一般固定 110-160px，其余内容列可留空或按比例给宽度。

## 2. 固定重复区和动态重复区

- Word 里写成“xxx 一 / xxx 二 / xxx 三”的分区，默认是固定分区，应该按原表顺序复制为多个 `report` 行段或多个顶层 `report`，不要自动变成 `subform`。
- 只有需求明确写了“可新增”“可删除”“任意多个”“多条明细”“按行增删”时，才使用 `subform` 或 `table`。
- `subform` 会渲染序号、添加、删除、分页等控制区。制式表格如果没有动态增删需求，`subform + report` 会破坏整体轮廓，产生蓝色按钮、编号标签、内缩边框。
- 如果确实需要动态重复块，但视觉必须像制式表格，优先评估 `table` 或关闭明显控制区；不要让动态控件占据 Word 原表没有的行列。

## 3. 组件选择要尊重 Word 符号

- Word 里的 `□`、`☑`、`○`、`●` 是选择控件信号，不是普通文本。
- 多个 `□` 并列通常转 `checkbox`，单组互斥选项通常转 `radio`。
- 不要把 Word 中横向排列的复选/单选项偷换成 `select`，除非用户明确要求下拉框或空间极度不足。
- 带“其他：____”的选项，保留“其他”选项并补一个同一行的输入框；不要把下划线直接当成普通文案。
- raw 样本里的 `radio` / `checkbox` 大多不依赖特殊 class；只有少量横向排布场景会额外使用 `radio-flex` 等样式类。
- 也就是说，默认优先把选项组本身放对，不要一开始就依赖 CSS 修复它们的横向布局。

## 4. 审批意见区

- 底部“审批意见”“xx意见”“签字”这类区域，默认是流程审批展示/签批区域。
- 如果用户没有明确要求表单发起人填写这些意见，不要把每一行都生成可编辑 `textarea` / `input`。
- 更稳的做法是按 Word 原样保留为空白表格行或静态签批占位；是否由流程引擎回填审批意见，以宿主平台能力和用户要求为准。
- 如果确实需要表单字段承载意见，必须说明这是业务字段，不是流程审批意见，并保持行高、边框和 Word 一致。

## 5. 色彩和边框控制

- Word 没有底色时，FormMaking 不要凭空加彩色背景；保持白底、灰色边框和宿主默认文本色。
- Word 有单元格底色时，只在对应分区标题或强调单元格上迁移底色；不要把整张表套成大面积彩色块。
- 正式表格常用 `borderColor: "#999"`、`borderWidth: 1`、`config.width: "850px"` 或 `"900px"`，并配合宿主 `newFormMaking` / `bord newFormMaking` 居中。
- `styleSheets` 只做必要增强，例如标题字号、分区标题居中、局部 padding；不要用 CSS 掩盖结构错误。
- 必填星号可以红，但标签正文、回填值、说明文字不应因为截图批注或 Word 标记而变红。

## 6. 行高和控件密度

- 单行输入控件放入 `report` 单元格时，优先 `hideLabel: true`、`width: "100%"` 或空宽度跟随单元格，不要让输入框比单元格窄一截。
- 大段文本行按 Word 行高设置 `textarea.rows` 或单元格 `height`，避免把长文本区压成普通一行。
- 文件上传放在原表对应单元格里；如果 Word 只是“上传附件”占位，不要额外拆出一整块蓝色上传区。

## 7. 交付前专项回扫

- 主表是否和标题同轴居中，宽度是否接近 Word。
- `report` 是否完整覆盖主体轮廓，是否漏了 `options.width = "100%"`。
- 固定“一/二/三”分区是否被误生成为动态 `subform`。
- Word 的复选/单选是否被误生成为下拉框。
- 审批意见区是否被过度字段化。
- 是否存在 Word 原表没有的蓝色新增按钮、编号列、删除按钮、分页控件。
